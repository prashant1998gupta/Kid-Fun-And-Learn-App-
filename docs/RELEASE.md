# Releasing KidVerse to the Play Store (Android)

Application id: `com.kidverse.kidverse`. The project is configured for a signed
release App Bundle; you only need to create your upload keystore once.

## 1. Create your upload keystore (once)
Keep this file safe and **never commit it** — losing it means you can't update
the app. From the repo root:

```bash
keytool -genkey -v -keystore kidverse-upload.jks \
  -keyalg RSA -keysize 2048 -validity 10000 -alias upload
```
Answer the prompts and choose a strong password. Move the `.jks` somewhere
outside the repo (e.g. `C:\Users\XRIG\keys\kidverse-upload.jks`).

## 2. Point the build at it
Create `android/key.properties` (already git-ignored):

```properties
storePassword=YOUR_STORE_PASSWORD
keyPassword=YOUR_KEY_PASSWORD
keyAlias=upload
storeFile=C:/Users/XRIG/keys/kidverse-upload.jks
```
`android/app/build.gradle.kts` auto-detects this file and signs the release
build with it. Without it, release builds fall back to debug keys (fine for
local testing, **not** accepted by Play).

## 3. Build the release App Bundle
```bash
flutter build appbundle
```
Output: `build/app/outputs/bundle/release/app-release.aab`.
(For sideloading a signed APK instead: `flutter build apk --release`.)

## 4. Bump the version each release
In `pubspec.yaml`, `version: 0.1.0+1` → the part after `+` is the Android
`versionCode` (must increase every upload); the part before is the user-facing
`versionName`.

## 5. Upload
Play Console → your app → Production (or Internal testing first) → create a
release → upload the `.aab`. Enroll in **Play App Signing** (recommended) so
Google manages the app signing key and your `.jks` is only the upload key.

## Play listing checklist (kids category / Families policy)
- Content rating questionnaire (target age; KidVerse is designed for under-13).
- **Privacy policy URL** (required, especially for Families) — the app is
  offline-first, collects no child PII; cloud sync/leaderboards are optional and
  parent-gated. See the About screen for the summary to base it on.
- Data safety form: declare optional account (email) + FCM token for sign-in
  families; no child data.
- Screenshots (phone + 7"/10" tablet), 512×512 icon, feature graphic.
- Target API level: current build targets a recent API and minSdk 24.
